# Fase 2 — El Panel: Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Panel funcional: gestión de notas (tabla filtrable + editor visual Tiptap), gestión de autores (firmas con foto), estados borrador/programada/publicada, con la estética "acreditación de prensa" de la Fase 1.

**Architecture:** Todo server-first: páginas server components que leen con el cliente de sesión (RLS decide qué puede ver/tocar cada rol), mutaciones vía Server Actions que revalidan. El editor Tiptap es la única isla client grande. Uploads de imágenes van del browser directo a Supabase Storage (bucket `imagenes`, RLS staff). El sitio público renderiza `cuerpo` (jsonb Tiptap → HTML server-side) con fallback al `contenido` text del mock.

**Tech Stack:** Next 16 Server Actions, Tiptap 3 (`@tiptap/react`, `starter-kit`, `extension-image`, `extension-link`, `extension-youtube`, `extension-placeholder`, `@tiptap/html` para render server), Supabase JS/SSR ya instalados.

## Dirección visual (frontend-design)

Extiende la "acreditación de prensa" del login a todo el panel — la metáfora es **la mesa de redacción del diario**:

- **Sidebar ink fija** (masthead vertical): logo badge arriba, secciones en Anton uppercase con tracking; item activo = barra roja 4px a la izquierda + texto blanco (inactivo: blanco 60%). Pie de sidebar: avatar/iniciales + email + rol + "Cerrar sesión".
- **Header de página**: overline mono roja (sección) + título Newsreader bold + acción primaria a la derecha (`.brut-cta-red`).
- **Tablas**: frame 2px ink + offset shadow 8px, header con fondo ink texto blanco mono, filas con hairline, hover fondo `--color-river-red-soft`.
- **Chips de estado**: `publicada` = fill rojo texto blanco · `programada` = fill ink texto blanco · `borrador` = outline ink sobre papel. Siempre mono uppercase 11px.
- **Forms**: inputs `.brut-frame` fondo blanco, focus borde rojo; labels `.brut-label`.
- **Sin rounded, sin blur, hover snap 120-160ms** — las reglas de siempre.

### Task 2: Tipos y capa de datos admin

### Task 3: Layout del panel (sidebar mesa de redacción)
- `app/admin/layout.tsx`: server component; obtiene user+profile (si falta profile → signout+login); renderiza `AdminSidebar` (nav + usuario + logout) y `<main>` con el contenido. El login queda FUERA del layout con sidebar (route group `app/admin/(panel)/` vs `app/admin/login/`).

### Task 4: `/admin/notas` — tabla filtrable
- Server component con `searchParams` (`estado`, `tipo`, `division`, `autor`, `q`).
- Toolbar de filtros (selects nativos estilizados + búsqueda) que hace GET.
- Tabla: título+slug, chip estado, tipo/división, firma, fecha, acciones (Editar / Publicar-Despublicar / Destacar / Borrar solo admin, con confirm).
- Editor (rol) ve todas pero solo puede tocar las suyas (RLS lo garantiza; la UI deshabilita).

### Task 5: Editor de nota `/admin/notas/nueva` y `/admin/notas/[id]`
- `npm i @tiptap/react @tiptap/pm @tiptap/starter-kit @tiptap/extension-image @tiptap/extension-link @tiptap/extension-youtube @tiptap/extension-placeholder @tiptap/html`

- Upload: `lib/admin/upload.ts` (client) — sube a `imagenes/posters/` o `imagenes/autores/` con nombre `slug-timestamp.ext`, devuelve URL pública.
